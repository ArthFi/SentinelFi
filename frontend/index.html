HOW IT WORKS ▸
  1. A CRE CronTrigger fires quarterly (Jan 1, Apr 1, Jul 1, Oct 1).
  2. Every DON node independently fetches the borrower’s financial report from the data source.
  3. Each node calls Gemini 2.0 Flash to extract DSCR and leverage ratio from unstructured text.
  4. consensusMedianAggregation takes the median across all nodes - Byzantine Fault Tolerant.
  5. The median metrics are ABI-encoded and written to LoanFacility.sol via the Keystone Forwarder.
  6. The contract checks covenants: if leverage > max OR DSCR < min, the loan is automatically frozen.
  7. This dashboard polls getAllLoans() every 5s to display the portfolio state in real-time.
